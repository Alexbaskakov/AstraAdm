# Лабораторная работа №2

## Знакомство с Ansible. Команды и сбор фактов

## Цель лабораторной работы
Освоить базовый синтаксис Ansible и научиться выполнять массовые операции на Linux-устройствах, включая проверку доступности, сбор системной информации (facts) и первичную настройку параметров с использованием ad-hoc команд.

## Задачи лабораторной работы
1. Установить Ansible на сервере управления.
2. Создать inventory-файл для Linux-устройств.
3. Настроить подключение к управляемым узлам по SSH-ключам.
4. Проверить доступность узлов с помощью Ansible.
5. Выполнить сбор системных фактов с Linux-устройств.
6. Выполнить массовую настройку базовых параметров (NTP, DNS) с помощью ad-hoc команд.
7. Ознакомиться с возможностью управления MikroTik через Ansible (без практического применения).

## Ожидаемый результат
После выполнения лабораторной работы студент умеет:
- устанавливать и проверять Ansible;
- создавать и использовать inventory-файл;
- выполнять ad-hoc команды;
- собирать факты о системе;
- применять базовые настройки сразу на нескольких узлах;
- понимать основные концепции Ansible.

# Теоретическая часть

**Ansible** — это инструмент автоматизации.  
Он позволяет **с одного сервера** выполнять команды **сразу на многих устройствах**.

Ключевые особенности:
- работает поверх SSH;
- не требует установки агентов на управляемые узлы;
- использует простые текстовые файлы (YAML, INI);
- подходит для автоматизации и администрирования.

### Основные понятия Ansible
- **Inventory** — файл со списком управляемых устройств.
- **Модули** — готовые действия (ping, setup, command, apt и др.).
- **Ad-hoc команды** — разовые команды без Playbook.
- **Факты (facts)** — информация о системе, которую Ansible собирает автоматически.

В данной лабораторной работе Ansible применяется **только для Linux-узлов**.  
Устройства MikroTik настраиваются вручную и используются лишь для ознакомления.

# 3. Создание inventory-файла

- Inventory-файл содержит список устройств, которыми управляет Ansible.

### 3.1 Создание файла hosts.ini

```
cat > hosts.ini <<EOF
[linux]
r1 ansible_host=10.0.12.1
r2 ansible_host=10.0.12.2
EOF
```

### 3.2 Проверка содержимого

```
cat hosts.ini
```

- Важно:

**linux** — имя группы;

**r1**, **r2** — логические имена узлов;

**ansible_host** — реальный IP-адрес устройства.

# 4. Настройка SSH-доступа по ключам

- Ansible использует SSH. Чтобы не вводить пароль каждый раз, применяются SSH-ключи.

### 4.1 Генерация SSH-ключа

```
ssh-keygen -t rsa -b 4096
```

На все вопросы можно нажимать **Enter** .

### 4.2 Копирование ключа на управляемые узлы

```
ssh-copy-id user@10.0.12.1
ssh-copy-id user@10.0.12.2
```

- Где user — имя пользователя на Linux-узле.

- Проверка:

```
ssh user@10.0.12.1
```

- Если вход без пароля — всё готово.

# 5. Проверка доступности узлов через Ansible

```
ansible linux -i hosts.ini -m ping
```

- Ожидаемый результат:

```
r1 | SUCCESS => {"ping": "pong"}
r2 | SUCCESS => {"ping": "pong"}
```

- Если есть ошибки — сначала исправить SSH-доступ.

# 6. Сбор системных фактов

- nAnsible может автоматически собирать информацию о системе.

```
ansible linux -i hosts.ini -m setup
```

### Будет выведена информация о:

- операционной системе;

- версии ядра;

- сетевых интерфейсах;

- процессоре;

- памяти.

Пример выборки одного факта

```
ansible linux -i hosts.ini -m setup -a "filter=ansible_hostname"
```

# 7. Выполнение ad-hoc команд

- Ad-hoc команды — это разовые действия без Playbook.

### 7.1 Проверка времени на всех узлах

```
ansible linux -i hosts.ini -m command -a "date"
```

# 8. Массовая настройка NTP
### 8.1 Установка chrony

```
ansible linux -i hosts.ini -m apt -a "name=chrony state=present update_cache=true" --become
```

### 8.2 Запуск и добавление в автозагрузку

```
ansible linux -i hosts.ini -m service -a "name=chrony state=started enabled=yes" --become
```

### 8.3 Проверка статуса

```
ansible linux -i hosts.ini -m command -a "systemctl status chrony --no-pager" --become
```

# 4. Настройка SSH-доступа по ключам

- Ansible использует SSH. Чтобы не вводить пароль каждый раз, применяются SSH-ключи.

### 4.1 Генерация SSH-ключа

```
ssh-keygen -t rsa -b 4096
```

На все вопросы можно нажимать **Enter** .

### 4.2 Копирование ключа на управляемые узлы

```
ssh-copy-id user@10.0.12.1
ssh-copy-id user@10.0.12.2
```

- Где user — имя пользователя на Linux-узле.

- Проверка:

```
ssh user@10.0.12.1
```

- Если вход без пароля — всё готово.

ansible linux -i hosts.ini -m ping

- Ожидаемый результат:

```
r1 | SUCCESS => {"ping": "pong"}
r2 | SUCCESS => {"ping": "pong"}
```

- Если есть ошибки — сначала исправить SSH-доступ.
