# Теоретическая часть

**GNS3** — это программа для моделирования компьютерных сетей.  
Она позволяет собирать виртуальные сети из маршрутизаторов, компьютеров и серверов без использования реального оборудования.

**Стенд** — это набор виртуальных устройств, соединённых между собой так же, как в реальной сети.

**Git** — система контроля версий.  
Она используется для хранения конфигураций, отслеживания изменений и возврата к предыдущему состоянию.

## 1. Создание проекта в GNS3

1. Запустить GNS3. (Желательно убедиться в настройках что адрес сервера выбран корректно: 10.242.192.200, порт 80, аутентификация выключена. Версия в окне Help - about: 2.2.43)
2. В меню выбрать **File → New Project**.
3. Указать имя проекта:

```bash
Lab01_GNS3_Stand
```

4. Выбрать папку для хранения проекта.
5. Нажать **OK**.

После этого откроется пустое рабочее поле.

---

## 2. Добавление образов устройств

### 2.1 Linux (Ubuntu Docker Guest — рекомендуется)

Ubuntu Docker Guest:

- быстро запускается;
- потребляет мало ресурсов;
- подходит для маршрутизаторов и ПК.

Добавление:

1. **File → Import Appliance**
2. Выбрать Ubuntu Docker Guest.
3. Завершить импорт.

---

### 2.2 MikroTik CHR

MikroTik CHR — виртуальная версия RouterOS.

Добавление:

1. **New QEMU VM**
2. Указать путь к образу `chr-*.qcow2`
3. Принять параметры по умолчанию.

---

## 3. Создание узлов стенда

Создать следующие устройства:

- Linux Router `R1`
- Linux Router `R2`
- ПК `PC1`
- ПК `PC2`
- Сервер управления `Mgmt`
- Узел `NAT` или `Cloud`

---

## 4. Построение базовой топологии

### 4.1 Подключение узлов

Соединения выполняются строго в следующем порядке:

1. `PC1:eth0` → `R1:eth0`  
Сеть: `10.0.1.0/24`

2. `R1:eth1` → `R2:eth0`  
Сеть: `10.0.12.0/24`

3. `R2:eth1` → `PC2:eth0`  
Сеть: `10.0.2.0/24`

4. `R2:eth2` → `Mgmt:eth0`  
Сеть управления: `10.0.99.0/24`

5. `Mgmt:eth1` → `NAT` или `Cloud`  
IP-адрес по DHCP

### 4.2 Логическая схема

```bash

PC1 ─── R1 ─── R2 ─── PC2
               │
              Mgmt
               │
           NAT / Cloud
```

---

## 5. Настройка IP-адресации

### 5.1 Назначение IP-адресов

**R1**
- eth0 — `10.0.1.1/24`
- eth1 — `10.0.12.1/24`

**R2**
- eth0 — `10.0.12.2/24`
- eth1 — `10.0.2.1/24`
- eth2 — `10.0.99.1/24`

**PC1**
- eth0 — `10.0.1.10/24`
- шлюз — `10.0.1.1`

**PC2**
- eth0 — `10.0.2.10/24`
- шлюз — `10.0.2.1`

**Mgmt**
- eth0 — `10.0.99.10/24`
- шлюз — `10.0.99.1`
- eth1 — DHCP

---

## 6. Настройка маршрутизации

```
# =========================
# R1
# =========================
ip addr add 10.0.1.1/24 dev eth0
ip addr add 10.0.12.1/24 dev eth1
ip route add 10.0.2.0/24 via 10.0.12.2
```

```
# =========================
# R2
# =========================
ip addr add 10.0.12.2/24 dev eth0
ip addr add 10.0.2.1/24 dev eth1
ip addr add 10.0.99.1/24 dev eth2
ip route add 10.0.1.0/24 via 10.0.12.1
```

## 7. Проверка сетевой связности

```
ping 10.0.2.10
ping 10.0.1.10
```
Если пакеты доходят — стенд собран правильно.

## 8. Установка и базовая настройка Git
Работа выполняется на сервере Mgmt.

- Установка Git
- 
```
sudo apt update
sudo apt install git -y
```

- Проверка версии
- 
```
git --version
```

## 9. Создание репозитория для конфигураций

- Создание каталога

```

cd network-configs
```

- Инициализация репозитория

```
git init
```

- Базовая настройка Git

```
git config user.name "Student"
git config user.email "student@example.com"
```

```

```
